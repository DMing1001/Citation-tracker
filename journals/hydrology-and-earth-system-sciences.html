收稿范围

Aims & Scope

HESS 是非营利性国际两阶段开放获取期刊，发表水文学领域的原创研究。鼓励基础研究和应用研究，推进对水文系统、水资源供给及水循环在地球系统功能中作用的理解。特别提倡多学科交叉方法。

  • 水循环过程：大陆水的物理、化学和生物过程（所有相态），从土壤水微观尺度到水文气候学全球尺度
  • 全球水资源：固、液、气态水资源的时空特征、储量、滞留时间、界面通量及各圈层间路径
  • 人水互动：人类活动对上述过程和通量的影响，以及可持续管理策略（洪水、干旱、荒漠化、土地退化、富营养化等）

涵盖的主题领域

  • Rivers and Lakes（河流与湖泊）
  • Coasts and Estuaries（海岸与河口）
  • Hydrometeorology（水文气象学）
  • Ecohydrology（生态水文学）
  • Water Resources Management（水资源管理）
  • Biogeochemical Processes（生物地球化学过程）

审稿流程

🌟 HESS 的互动评审流程不同于传统期刊！整个过程分为提交→初筛→公开讨论→最终修订四个阶段，其中公开讨论阶段是最独特的环节。

审稿制度

评审方式
公开互动评审

6 周（EGUsphere）

完整流程时间线

  1. 提交与编辑分配 → 稿件提交至 Copernicus Editor 系统，分配到相关学科领域的编辑
  2. Access Review（初筛） → 编辑评估稿件是否在期刊范围内、基本科学质量是否达标、是否对水文学有新贡献。可建议技术性修改（排版错误、图表澄清等），但此阶段不得要求修改科学内容
  3. 公开讨论（6 周） → 通过初筛后，稿件以预印本形式发布在 EGUsphere，获得 DOI 并可被引用：
    • 指定审稿人（可选择匿名或实名）提交审稿意见
    • 科学界其他成员（实名）可参与讨论
    • 作者应积极参与——尽早回复审稿人和社区评论以激发进一步讨论
    • 通常每篇预印本至少收到 2 份审稿意见
  4. 最终回复 → 公开讨论结束后，作者须在 4 周内发布对所有评论的回应（若未在讨论期间完成）
  5. 编辑决定 → 编辑基于作者对所有评论的回应决定：邀请修订 / 直接录用 / 拒稿。必要时可继续传统同行评审
  6. 正式发表 → 录用后排版校对，在 HESS 网站发表。文章附带此前预印本和互动讨论的链接，所有审稿报告、编辑意见、作者回应及不同版本稿件均公开

评审标准

编辑和审稿人依据 HESS Review Criteria 评估稿件，重点关注：科学原创性、方法严谨性、结果清晰度、对水文学的贡献。

投稿流程

投稿步骤

  1. 下载 Copernicus 稿件模板（LaTeX 或 MS Word），或使用 R Markdown
  2. 按要求准备 PDF 稿件：含行号、页码，单栏格式
  3. 准备摘要文本和短摘要（Short Summary，500 字符以内非技术性文本，用于推广）
  4. 选择合适的文章类型
  5. 注册稿件 → 收到上传链接进入 Copernicus Office Editor
  6. 上传稿件文件（PDF，≤ 50 MB）
  7. 定义学科领域和索引词（用于编辑分配）
  8. 确认提交
📌 重要提醒：HESS 不需要传统的 Cover Letter。取而代之的是 Short Summary（500 字符以内的非技术性推广文本）和学科领域选择。

AI 工具使用

如使用 AI 工具辅助生成稿件的部分内容，须在 Methods 部分或 Acknowledgements 中描述使用情况。

投稿后注意事项

  • 可通过 Copernicus 系统实时追踪稿件状态
  • 初筛（Access Review）通常较快，编辑可建议技术性修改后再发布预印本
  • 公开讨论期间作者应积极回应评论——这是 HESS 流程的核心
  • 修订稿在讨论结束后 4 周内提交
  • 从接收到正式发表通常 1–2 周（排版校对）

稿件准备

文件格式

  • 评审阶段：提交 PDF 文件（嵌入字体，单栏，竖版，含行号和页码）
  • 录用后排版：LaTeX（.tex）或 Word（.doc/.docx）
  • 也可使用 R Markdown 格式提交
  • 图分辨率 ≥ 300 dpi，格式：PDF / PS / EPS / JPG / PNG / TIFF
  • 文件大小 ≤ 50 MB

文章结构

  1. Title Page — 标题（简洁，避免缩写）、作者全名、完整单位地址、通讯作者邮箱。已故合著者需标注
  2. Abstract — 普通读者可理解的摘要，简要介绍主题后总结关键点并展望未来研究方向。避免引用和未解释的缩写
  3. Graphical Abstract（可选） — 作为 Key Figure 上传，或作为未编号图片放在摘要之后
  4. Introduction
  5. Sections — 所有节标题必须编号（如 3, 3.1, 3.1.1），三级标题以内。正文中使用 "Sect." 缩写
  6. Conclusions
  7. Appendices — 非核心但有用的补充信息，用大写字母标注（Appendix A, B…）
  8. Code Availability — 代码可获取性声明
  9. Data Availability — 数据可获取性声明
  10. Author Contribution — 推荐使用 CRediT 标准
  11. Competing Interests — 利益冲突声明
  12. Acknowledgements — 致谢与基金信息
  13. References
💡 Copernicus 模板优势：HESS 提供官方 LaTeX 和 Word 模板，直接使用模板即可满足大部分格式要求（单栏、节编号等），不需要预排双栏格式。
📐

格式要求

图表规范

  • 图格式：PDF / PS / EPS / JPG / PNG / TIFF，分辨率 ≥ 300 dpi
  • 面板图合并为一个文件，编号阿拉伯数字（fig01, fig02…）
  • 图表标题须独立可读（不依赖正文）
  • 彩色图免费（在线版）
  • 须确保色盲友好（Colour Vision Deficiency friendly）
  • 地图和航拍图须包含版权声明或出处标注
  • 图表插入正文中首次引用位置附近（不要放在文末）

补充材料

  • 非核心补充材料以 ZIP 或单独 PDF 提交（≤ 50 MB）
  • 更大的数据集/模型代码/视频应上传至可靠数据仓库获取 DOI
  • 推荐：Zenodo、Figshare 等
  • 补充材料在正文中引用并纳入参考文献列表

数据与代码政策

  • 强烈建议将数据集提交至可靠数据仓库并获取 DOI
  • Code Availability 和 Data Availability 为必填声明
  • 软件/模型代码建议上传至 GitHub 并通过 Zenodo 存档

Short Summary（必填）

  • 500 字符以内（含空格）的非技术性英文段落
  • 用于在社交媒体和期刊推广中介绍你的工作
  • 需包含：主要结论、研究意义、简要背景和方法
  • 避免缩写，使用完整术语
📖

参考文献

引用格式

采用 Copernicus 编号制，文中方括号引用：

  • 单篇：[1]
  • 多篇：[1,3,5] 或 [1–5]
  • 按引用顺序编号

参考文献格式示例

期刊文章：

Beven, K. and Kirkby, M. J.: A physically based, variable contributing area model of basin hydrology, Hydrol. Sci. Bull., 24, 43–69, https://doi.org/10.1080/02626667909491834, 1979.


书籍：

Chow, V. T., Maidment, D. R., and Mays, L. W.: Applied Hydrology, McGraw-Hill, New York, ISBN 0-07-010811-4, 1988.


书籍章节：

Sivapalan, M.: Process complexity at hillslope scale, process simplicity at the watershed scale: is there a connection?, Hydrol. Process., 17, 1037–1041, https://doi.org/10.1002/hyp.5109, 2003.

格式要点

  • 作者姓名：名在前姓在后，用 "and" 连接最后两位作者（Beven, K. and Kirkby, M. J.）
  • 期刊名使用标准缩写
  • 按引用顺序排列（非字母序）
  • DOI 为必填项（如有）
  • URL 拼写完整，不要超链接嵌入文字

投稿清单

提交前检查以下事项

  • 稿件符合 HESS Aims & Scope
  • 文章类型选择正确（标题格式符合要求）
  • 使用 Copernicus 官方 LaTeX 或 Word 模板
  • PDF 稿件含行号和页码，单栏竖版
  • 所有节标题编号正确（3, 3.1, 3.1.1）
  • 摘要不含未解释的缩写和引用
  • Short Summary（500 字符非技术性文本）已准备
  • 作者全名和完整单位信息
  • Author Contribution 声明已填写（推荐 CRediT）
  • Competing Interests 声明已提供
  • Code Availability 和 Data Availability 声明
  • 图表在正文中首次引用位置附近，色盲友好
  • 图片分辨率 ≥ 300 dpi
  • 参考文献为编号制，含 DOI
  • 补充材料以 ZIP/PDF 提交，或上传至数据仓库获取 DOI
  • 如使用 AI 工具，已在 Methods 或 Acknowledgements 中声明
  • 地图/航拍图含版权标注
  • 英文语言质量良好

伦理与政策

利益冲突 (Competing Interests)

所有作者必须声明利益冲突。若无冲突，须声明："The authors declare that they have no conflict of interest."

详情参见 Competing Interests Policy。

AI 使用政策

使用 AI 工具辅助写作须在 Methods 部分或 Acknowledgements 中声明使用范围和方式。AI 不能列为作者。详情参见 Publication Policy。

数据政策

强烈建议将数据集上传至可靠公开仓库并获取 DOI，在稿件中引用。详见 Data Policy。

许可与版权

HESS 采用 CC BY 4.0 开放许可，作者保留版权，允许任何人共享和再利用（需注明出处）。详见 Licence & Copyright。
